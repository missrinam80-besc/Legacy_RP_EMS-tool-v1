# README-richtlijnen

## Doel

Elke module krijgt een eigen `README.md` zodat de werking van die module los van de rest begrepen en onderhouden kan worden.

---

## Verplichte onderdelen

### 1. Titel en korte beschrijving
Leg kort uit wat de module doet.

### 2. Doel
Omschrijf waarvoor de module dient.

### 3. Functionaliteiten
Geef een overzicht van de belangrijkste functies.

### 4. Bestandsstructuur
Toon welke bestanden in de module zitten en wat hun rol is.

### 5. Werking
Beschrijf in grote lijnen hoe invoer verwerkt wordt naar output.

### 6. Configuratie
Leg uit welke gegevens via JSON of instellingen beheerd worden.

### 7. Gebruik
Beschrijf hoe de module gebruikt wordt.

### 8. Bekende beperkingen
Wees duidelijk over wat nog niet ondersteund wordt.

### 9. Toekomstige uitbreidingen
Beschrijf mogelijke vervolgstappen.

## Richtlijnen voor schrijfstijl

- schrijf helder en concreet
- gebruik korte secties
- vermijd vage beschrijvingen
- benoem zaken zoals ze in de code voorkomen
- maak onderscheid tussen functionele uitleg en technische uitleg
